# Domain Design

Unit-level domain and logical design documents live here.

Use `domain_design_TEMPLATE.md` for technology-agnostic domain modeling. Domain design should define business concepts, aggregates, entities, value objects, domain services, invariants, events, and traceability back to Units without choosing a database, framework, cloud provider, or programming language.

Use `logical_design_TEMPLATE.md` after the domain model is approved. Logical design may document component boundaries, runtime responsibilities, NFR-driven architecture decisions, integration behavior, failure handling, security constraints, and non-binding technology mapping.
