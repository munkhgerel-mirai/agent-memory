# Release Notes: Copier Answers Generation Fix

**Version:** v0.2.1  
**Status:** Prepared for the managed-template patch release  

## Overview

This patch fixes Newly Adopting Project brownfield adoption so Copier writes `.copier-answers.yml` during `copier copy`.

## Fixed

- Adds the Copier answers template file so `.copier-answers.yml` is rendered into destination projects.
- Keeps `.copier-answers.yml` as template-owned sync metadata.
- Updates canonical adoption and upgrade examples to use `v0.2.1`.

## Known Issue In v0.2.0

`v0.2.0` configured the answers file name but did not include the rendered answers template. Projects copied from `v0.2.0` may have `.ai-dlc-template.yml` but no `.copier-answers.yml`, which prevents later `copier update` runs.

## Repair Guidance

Preferred repair path after this release:

```bash
copier copy --vcs-ref v0.2.1 https://github.com/miraitechnologies/ai_dlc_template.git .
```

If a project must repair metadata before re-copying, create `.copier-answers.yml` temporarily with project-specific values:

```yaml
_src_path: https://github.com/miraitechnologies/ai_dlc_template.git
_commit: v0.2.0

project_name: "YOUR_PROJECT_NAME"
project_intent: "YOUR_PROJECT_INTENT"
template_source: "https://github.com/miraitechnologies/ai_dlc_template.git"
template_version: "v0.2.0"
template_commit: "v0.2.0"
```

Manual creation is a repair workaround only. New adoptions should use Copier so the answers metadata is generated from the template.
