# Release Notes: Managed AI-DLC Template Governance

**Version:** v0.2.0  
**Status:** Prepared for the managed-template governance release  

## Overview

This release turns the AI-DLC template into a versioned managed template that can be adopted by new repositories and upgraded by existing repositories without overwriting project-owned work.

## Highlights

- Adds explicit template-owned, project-owned, and manual-merge file policy.
- Preserves downstream `README.md` during upgrades and brownfield adoption.
- Preserves downstream `.gitignore` during upgrades and brownfield adoption.
- Adds Copier-first metadata for template source and applied version tracking.
- Adds a sync wrapper that blocks project-owned file changes and reports `.gitignore` recommendations.
- Adds an upgrade guide for Template-Adopted Projects and brownfield adoption guide for Newly Adopting Projects.
- Refines session log ownership so `session-logs/README.md` is template-owned while real session log entries remain project-owned.

## Governance

- `README.md`, `.gitignore`, `PROJECT_STATUS.md`, real session log entries, real source code, real tests, approved plans, and generated non-template lifecycle artifacts are project-owned.
- `session-logs/README.md` is template-owned.
- `AGENTS.md` is manual-merge.
- Template upgrades must fail if they modify project-owned files without explicit approval.
- `.gitignore` changes are recommendations only after initial project creation.

## Upgrade Notes

- Template-Adopted Projects should add template metadata, run the sync wrapper, resolve manual merges, and run `ai-dlc-setup` in audit mode.
- Newly Adopting Projects should preserve existing project files, add the AI-DLC governance layer, run `ai-dlc-setup`, then run `ai-dlc-brownfield-discovery` before design or implementation changes.
